* `add_child`를 호출했는데 곧바로 `get_parent()`가 `null`을 반환한다면, 실제로 child 추가가 실패했는지 먼저 봐야 한다.
  * 에디터에서 breakpoint를 걸고 확인하면 아래처럼 parent node가 children을 설정 중이라 추가가 실패했다는 오류가 보일 수 있다.

```text
E 0:00:27.497   add_child: Parent node is busy setting up children, add_node() failed. Consider using call_deferred("add_child", child) instead.
  <C++ 오류>      Condition "data.blocked > 0" is true.
  <C++ 소스>      scene/main/node.cpp:1135 @ add_child()
  <스택 추적>       map.gd:80 @ generate_tiles()
                dungeon_generation.gd:66 @ _ready()
```

* 이 경우에는 `parent.call_deferred("add_child", child)`를 사용한다.
  * 현재 node tree가 setup 중인 시점에는 바로 child를 추가할 수 없고, deferred call로 다음 idle step에 실행되도록 미루면 된다.
* 핵심은 `get_parent()`가 이상한 값을 돌려준 것이 아니라, `add_child()` 자체가 실패했다는 점이다.
  * Godot lifecycle 중 `_ready()`에서 node tree를 변경할 때 이런 문제가 생길 수 있다.
  * parent/child 관계가 기대와 다르면 먼저 editor output의 에러 로그를 확인하는 편이 빠르다.
