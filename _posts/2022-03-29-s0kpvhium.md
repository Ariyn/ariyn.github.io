Rust의 소유권 규칙은 세 가지로 요약할 수 있다.
값은 owner를 하나 가진다.

{: .tractatus}
1. Rust의 소유권 규칙은 세 가지로 요약할 수 있다.

   1. 값은 owner를 하나 가진다.

   2. 한 시점에 owner는 하나뿐이다.

   3. owner가 스코프를 벗어나면 값은 drop된다.

2. 할당에서 헷갈리는 지점은 `String` 같은 heap 데이터를 가진 값이다.

   ```rust
       let x = String::from("hello");
       let y = x;
   ```

   1. 이 코드는 `x`를 `y`로 복사해서 owner가 둘이 되는 코드가 아니다.

   2. `String` 값의 소유권이 `x`에서 `y`로 이동(move)한다.

   3. 그래서 `x`는 더 이상 사용할 수 없고, 마지막에는 `y`만 drop된다.

   4. Rust가 이렇게 하는 이유는 같은 heap 버퍼를 두 owner가 동시에 drop하는 double free를 막기 위해서다.

3. 이동된 값을 다시 사용하면 컴파일 에러가 난다.

   ```rust
         let x = String::from("hello");
         let y = x;

         println!("{}", x);
         error[E0382]: use of moved value: `s1`
          --> src/main.rs:4:27
           |
         3 |     let y = x;
           |         -- value moved here
         4 |     println!(x);
           |              ^^ value used here after move
           |
           = note: move occurs because `x` has type `std::string::String`,
         which does not implement the `Copy` trait
   ```

   1. `i32`처럼 `Copy` trait을 구현한 타입은 대입해도 기존 변수를 계속 쓸 수 있다.

   2. `Copy`는 비트 단위 복사가 안전한 타입에 붙는 성격이고, `Drop`과 동시에 구현할 수 없다.

   3. 명시적으로 데이터를 복제하고 싶으면 `clone()`을 호출해야 한다.

4. 함수 호출도 대입과 같은 소유권 이동 규칙을 따른다.

   ```rust
       let x = String::from("hello");
       sample(x); // x가 파라미터 s로 move됨
       println!(x); // 소유권이 종료되었기 때문에 사용 불가

       fn sample(s: String) {
         println!(s);
       } // s의 소유권 종료. s의 drop 호출
   ```

   1. `sample(x)`를 호출하면 `x`의 소유권이 매개변수 `s`로 이동한다.

   2. 함수가 끝나면 `s`가 drop되므로 호출한 쪽에서는 `x`를 더 이상 사용할 수 없다.

5. 해결 방법은 목적에 따라 다르다.

   1. 값을 읽기만 한다면 `&String`이나 `&str`로 빌린다.

   2. 값을 수정해야 한다면 `&mut String`으로 빌린다.

   3. 함수가 값을 소비하고 다시 돌려줘야 한다면 반환값으로 소유권을 다시 이동시킨다.

   4. 독립적인 복사본이 필요하면 `clone()`을 명시한다.
