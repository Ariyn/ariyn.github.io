* API Gateway에서 Integration Request의 header mapping을 설정할 때 아래 오류가 날 수 있다.
  * `Invalid mapping expression specified: Validation Result: warnings : [], errors : [Invalid mapping expression specified: method.request.header.Authorization]`
* 이 경우 Method Request 단계에 해당 header가 먼저 정의되어 있는지 확인한다.
  * 예를 들어 integration mapping에서 `method.request.header.Authorization`을 쓰려면, Method Request의 Headers에도 `Authorization`이 있어야 한다.
  * Integration Request가 임의의 method request 값을 마음대로 참조할 수 있는 구조가 아니다.
* 순서는 Method Request에서 받을 값을 선언하고, Integration Request에서 그 값을 backend로 넘기는 식이다.
  * 설정 화면이 나뉘어 있어서 header를 한쪽에만 넣고 놓치기 쉽다.
