* Docker registry를 거치지 않고 이미지를 다른 컴퓨터로 옮기려면 `docker save`와 `docker load`를 사용할 수 있다.
  * `docker save`는 로컬 image를 tar 파일로 내보낸다.
  * `docker load`는 tar 파일을 다시 Docker image로 불러온다.

```shell
docker save -o image.tar <image name>
docker load -i image.tar
```

* 네트워크가 막힌 환경이나 임시 전달에는 편하다.
  * 다만 팀이나 운영 환경에서는 registry에 push/pull 하는 방식이 추적과 배포 관리에 더 낫다.
* 옮긴 뒤에는 image tag가 기대대로 남아 있는지 확인한다.
  * 필요하면 `docker tag`로 새 이름을 붙여서 사용한다.
