* Python에서 직접 만든 클래스를 비교 가능하게 만들려면 rich comparison method를 구현한다.
  * 대표적으로 `__eq__`, `__lt__`, `__le__`, `__gt__`, `__ge__`가 있다.
  * `heapq`, `sorted`, `list.sort`처럼 객체의 순서를 비교해야 하는 곳에서 필요하다.
* 비교 기준은 한 곳으로 모아두는 편이 낫다.
  * 예를 들어 `Job` 객체를 `elapse_time - request_at` 기준으로 비교한다면, 그 값을 반환하는 helper를 두고 각 비교 메서드에서 재사용한다.
  * 이렇게 해야 비교 기준이 바뀌어도 수정 위치가 줄어든다.

```python
class Job:
    def __init__(self, request_at, elapse_time):
        self.request_at = request_at
        self.elapse_time = elapse_time

    def value(self):
        return self.elapse_time - self.request_at

    def __lt__(self, other):
        return self.value() < other.value()

    def __le__(self, other):
        return self.value() <= other.value()

    def __gt__(self, other):
        return self.value() > other.value()

    def __ge__(self, other):
        return self.value() >= other.value()

    def __eq__(self, other):
        return self.value() == other.value()
```

* 주의할 점은 각 연산자의 방향을 정확히 맞추는 것이다.
  * `__ge__`는 `>=` 비교를 해야 한다.
  * `__gt__`, `__ge__`까지 직접 구현하면 복사해서 붙이는 과정에서 부호를 잘못 넣기 쉽다.
* 실제 코드에서는 `functools.total_ordering`도 고려할 수 있다.
  * `__eq__`와 `__lt__`처럼 최소 비교 연산만 직접 구현하면 나머지 순서 비교를 채워준다.
  * 다만 성능이나 명시성이 중요하면 필요한 메서드를 직접 구현하는 쪽이 더 분명할 때도 있다.
