* GitLens는 VS Code에서 Git 정보를 코드 옆에 보여주는 확장이다.
  * 특정 라인을 마지막으로 바꾼 커밋, 작성자, 커밋 메시지를 바로 볼 수 있다.
  * `git blame`을 editor 안에서 계속 확인할 수 있게 해주는 도구라고 보면 된다.
* 코드를 읽을 때 "왜 이렇게 되었는지"를 따라가기가 쉬워진다.
  * 변경 이력, 관련 커밋, 이전 diff를 빠르게 열 수 있어서 맥락 파악에 도움이 된다.
* IntelliJ 계열 IDE도 비슷한 기능을 제공한다.
  * 기본 annotate 기능이나 Git 관련 플러그인을 확인하면 된다.
  * 도구 이름보다 중요한 것은 라인 단위 변경 맥락을 코드 읽기 흐름 안에서 바로 보는 것이다.
