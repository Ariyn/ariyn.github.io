GitHub 저장소를 HTTPS 대신 SSH로 접근하게 만들고 싶다면 Git config의 `insteadOf`를 사용할 수 있다.
`https://github.com/`으로 시작하는 URL을 자동으로 `ssh://git@github.com/` 형태로 바꾸는 방식이다.

{: .tractatus}
1. GitHub 저장소를 HTTPS 대신 SSH로 접근하게 만들고 싶다면 Git config의 `insteadOf`를 사용할 수 있다.

   1. `https://github.com/`으로 시작하는 URL을 자동으로 `ssh://git@github.com/` 형태로 바꾸는 방식이다.

   2. 매번 clone URL을 고치지 않아도 되기 때문에 여러 저장소를 다룰 때 편하다.

      ```gitconfig
      [url "ssh://git@github.com/"]
          insteadOf = https://github.com/
      ```

2. 전역으로 적용하려면 global config에 추가한다.

   1. `git config --global url."ssh://git@github.com/".insteadOf https://github.com/`

   2. 적용 후에는 SSH key가 GitHub에 등록되어 있어야 실제 접속이 된다.

3. 이 설정은 편하지만, 모든 GitHub HTTPS URL에 영향을 준다.

   1. 특정 프로젝트에서만 쓰고 싶다면 repository-local config에 넣는 편이 낫다.
