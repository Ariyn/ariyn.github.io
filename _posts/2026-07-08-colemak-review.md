Colemak 전환은 입력 경험을 완전히 바꾸는 사건은 아니었다.
다만 코드 입력량이 줄어든 시점에는 충분히 시도해볼 만한 생활 도구 조정이었다.

{: .tractatus}
1. 한국어 입력은 이미 오래전에 세벌식으로 바꿔두었다.

   1. 세벌식은 10년째 쓰고 있었다.

   2. 그래서 키보드 배열을 새로 배우는 일 자체에는 거부감이 크지 않았다.

   3. 반면 영문 입력은 계속 QWERTY에 남아 있었다.

   4. 언젠가는 다른 영문 배열도 써보고 싶었고, 이번에는 Colemak을 골랐다.


2. 지금이 전환하기 좋은 시기라고 본 이유는 코드 입력량이 줄었기 때문이다.

   1. AI 도구를 쓰면서 예전처럼 모든 코드를 직접 치는 시간이 줄었다.

   2. 영문 배열 전환으로 잠깐 느려져도 업무 전체가 크게 무너지지는 않을 것 같았다.

   3. 예전이었다면 당장 생산성이 떨어지는 부담 때문에 시도하기 어려웠을 것이다.


3. 전환할 때는 QWERTY와 애매하게 병행하지 않기로 했다.

   1. OS의 영문 배열을 완전히 Colemak으로 바꿨다.

   2. 키보드에는 유성 매직으로 Colemak 위치를 직접 적어두었다.

   3. 처음에는 키보드를 보면서 치더라도 손이 QWERTY로 돌아가지 못하게 했다.

   4. 배열 전환에서는 의지보다 환경 고정이 더 중요했다.


4. 4주 정도 지난 지금은 영어 문장을 키보드 없이 볼 수 있게 되었다.

   1. 아직 아주 자연스럽게 빠른 상태는 아니다.

   2. QWERTY와 헷갈리는 키가 남아 있다.

   3. 자주 치지 않던 영어 문장은 속도가 떨어진다.

   4. 머릿속에 단어가 익숙하지 않으면 배열까지 같이 떠올려야 해서 더 느려진다.


5. 편해진 부분은 분명히 있다.

   1. 대부분의 영단어를 가운뎃 줄 중심으로 칠 수 있다.

   2. 손가락이 위아래로 크게 움직이는 일이 줄었다.

   3. 긴 문장을 칠 때 부담이 덜하다.

   4. QWERTY에서 당연하게 받아들이던 손가락 이동이 꽤 많았다는 것을 알게 된다.


6. 그렇다고 입력 경험 전체가 드라마틱하게 바뀐 것은 아니다.

   1. 세벌식을 오래 쓰면서도 비슷한 생각을 했다.

   2. 배열을 바꾸면 편해지는 부분은 분명히 있다.

   3. 하지만 배열만으로 속도와 정확도가 완전히 달라지지는 않는다.

   4. 실제 입력 경험은 사용 시간, 자주 쓰는 문장, 도구 환경의 영향을 많이 받는다.


7. 현재 결론은 Colemak의 기대값을 낮게 잡아야 한다는 것이다.

   1. 새로운 배열을 배우는 재미는 있다.

   2. 홈 로우 중심 입력은 확실히 편하다.

   3. 당장 생산성이 중요한 시기에 무리해서 바꿀 정도의 변화는 아니다.

   4. 입력 방식을 바꾸는 일은 혁신이라기보다 생활 습관을 다시 맞추는 일에 가깝다.
