* Rust의 lifetime은 참조가 유효한 범위를 컴파일러에게 설명하는 표기다.
  * 소유권 자체의 생존 기간이라기보다, 빌린 값의 참조가 언제까지 안전한지를 나타낸다.
  * 대부분은 lifetime elision 규칙 때문에 직접 쓰지 않아도 된다.
  * 다만 함수가 여러 참조를 받고 그중 하나를 다시 반환하는 경우처럼 관계가 애매하면 명시해야 한다.
* 아래 함수에서 `'a`는 `x`, `y`, 반환값이 모두 같은 기간을 산다는 뜻이 아니다.
  * 반환값은 `x`와 `y` 중 더 짧게 살아있는 참조보다 오래 사용할 수 없다는 제약에 가깝다.
  * 그래서 `longest`가 어느 쪽을 반환하더라도 dangling reference가 생기지 않는다.
  ```rust
  fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() {
      x
    } else {
      y
    }
  }
  
  fn main() {
    let result;
    let x = "123";
    {
      let y = "1";
      result = longest(x, y);
  
      println!("result = {}", result);
    }
  
    println!("result = {}", result); // <<<< compile error.
    // result는 x나 y 중 더 짧은 lifetime 밖에서 사용할 수 없다.
    // 이 예제에서는 y가 안쪽 블록에서 끝나므로, 블록 밖의 result 사용은 거부된다.
  }
  ```
