* Rust에서 trait을 구현할 때는 `impl Trait for Type` 형태를 사용한다.

```rust
impl TraitName for StructName {
    fn method(&self) {
        // ...
    }
}
```

* trait은 "이 타입이 어떤 동작을 제공한다"는 계약을 표현한다.
  * 함수가 특정 trait을 요구하면, 그 trait을 구현한 타입만 받을 수 있다.
  * Java나 Go의 interface와 비슷하게 볼 수 있지만, Rust의 ownership과 generic 시스템 안에서 동작한다.
* trait에는 default implementation을 둘 수도 있다.
  * trait 선언부에서 함수 body까지 제공하면, 구현 타입이 그 메서드를 생략해도 기본 동작을 사용한다.

```rust
pub trait Sample {
    fn sample() -> String {
        String::from("Hello")
    }
}

struct S;

impl Sample for S {}
```

* 필요한 타입만 기본 구현을 덮어쓸 수 있다.
  * 공통 동작은 trait에 두고, 특수한 타입만 직접 구현하면 중복을 줄일 수 있다.
