* Git에서 untracked file을 지울 때는 `git clean -f`를 사용한다.
  * 추적 중이지 않은 파일만 삭제하고, Git이 추적하는 변경사항은 건드리지 않는다.
  * 디렉터리까지 지우려면 `git clean -fd`를 사용한다.
* 먼저 무엇이 지워질지 확인하고 싶다면 `git clean -n`으로 dry run을 해보는 것이 좋다.
  * 실제 삭제 전에 대상 목록을 볼 수 있다.
  * 실수로 생성 파일이나 로컬 메모를 지우는 일을 줄일 수 있다.
* 이 명령은 삭제 후 복구가 까다롭다.
  * 특히 untracked file은 Git 히스토리에 없으므로, 실행 전에 한 번 더 확인해야 한다.
