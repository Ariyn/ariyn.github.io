* Jenkins에서 GitHub private repository push trigger가 동작하지 않으면 먼저 Jenkins 로그를 확인한다.
  * GitHub webhook 요청이 Jenkins까지 들어오는지 봐야 한다.
  * 요청 자체가 안 보이면 GitHub webhook URL, 네트워크, Jenkins endpoint 설정 문제일 가능성이 크다.
* 로그에 `no matching repository`가 보이면 repository URL 매칭을 확인한다.
  * Jenkins GitSCM에 등록된 URL과 webhook payload에 들어온 repository URL이 같은 형식인지 봐야 한다.
  * private repository를 SSH로 연결한 경우 `sshUrl`, 사용자 이름, `.git` suffix 차이 때문에 매칭이 깨질 수 있다.
* 해결은 Jenkins에 등록된 repository URL을 webhook payload의 URL과 맞추는 쪽이다.
  * 같은 저장소라도 `git@github.com:user/repo.git`, `ssh://git@github.com/user/repo.git`, `https://github.com/user/repo`는 서로 다른 문자열로 취급될 수 있다.
  * webhook 문제는 권한보다 URL 매칭 문제인 경우가 있으니 로그를 먼저 보는 것이 빠르다.
