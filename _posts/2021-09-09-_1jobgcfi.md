이럴 때는 필드 이름을 문자열로 감싸서 `."$id"`처럼 접근한다.

{: .tractatus}
1. `jq`에서 필드 이름이 `$`로 시작하면 dot notation으로 바로 접근하기 어렵다.

   1. 이럴 때는 필드 이름을 문자열로 감싸서 `."$id"`처럼 접근한다.

   2. 예시는 `jq '."$id"'`다.

2. JSON key는 문자열이기 때문에, 특수문자가 들어간 key는 명시적으로 문자열 접근을 하는 쪽이 안전하다.

   1. `jq '.foo'`처럼 쓸 수 있는 것은 key가 일반 식별자처럼 해석될 수 있을 때다.

   2. `$id`, `foo-bar`, `some.key` 같은 이름은 `."..."` 형태가 더 분명하다.

3. API 응답에서 JSON Schema의 `$id`, `$ref` 같은 필드를 다룰 때 자주 필요하다.
