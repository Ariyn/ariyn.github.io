* Scala에서는 인자가 없는 메서드를 괄호 없이 선언할 수 있다.
  ```scala
  abstract class Element {
    def contents: Array[String]
    def height: Int = contents.length
    def width: Int =
      if (height == 0) 0 else contents(0).length
  }
  
  val element: Element = ???
  val height = element.height
  ```

  * 출처: [Programming in Scala, Composition and Inheritance](https://www.artima.com/pins1ed/composition-and-inheritance.html#i1343251059-1:~:text=Listing%2010.2%20%2D%20Defining%20parameterless%20methods%20width%20and%20heigh)
* `def foo: Int`는 parameterless method이고, `def foo(): Int`는 empty-paren method라고 부르는 것 같다.
  * 참고: https://stackoverflow.com/questions/7600910/difference-between-function-with-parentheses-and-without
  * 둘 다 인자는 없지만 호출 관례가 다르다.
* 관례상 사이드 이펙트가 없는 값 계산은 괄호 없이, 사이드 이펙트가 있는 동작은 괄호를 붙이는 편이 자연스럽다.
  ```scala
    "hello".length  // no () because no side-effect
    println()       // better to not drop the ()
  ```

  * 참고: https://softwareengineering.stackexchange.com/questions/235925/parameterless-empty-paren-methods-in-scala
* 기억할 점은 "괄호를 생략할 수 있다"가 전부가 아니라, API를 읽는 사람에게 이것이 값처럼 보이는 계산인지, 호출마다 어떤 동작을 하는 메서드인지 힌트를 준다는 점이다.
