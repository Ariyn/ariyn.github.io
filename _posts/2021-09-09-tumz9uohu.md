* Docker 컨테이너에 호스트 파일을 넘길 때는 bind mount를 사용한다.
  * 기본 형태는 `-v <host-path>:<container-path>`다.
  * 호스트 경로는 절대 경로로 쓰는 편이 안전하다.

```shell
docker run -v "$PWD/file.txt:/root/file.txt" <image>
```

* 상대 경로는 실행 위치와 Docker 해석 방식에 따라 헷갈릴 수 있다.
  * 그래서 `$PWD`나 절대 경로를 사용하면 의도한 파일이 명확해진다.
* 파일 하나를 mount할 때는 대상 경로가 파일인지 디렉터리인지도 확인해야 한다.
  * 경로를 잘못 쓰면 컨테이너 안에 예상과 다른 디렉터리가 생겨 디버깅이 귀찮아진다.
