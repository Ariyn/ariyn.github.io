* Rust의 binding은 기본적으로 immutable이다.
  * 값을 나중에 바꿔야 한다면 선언할 때 `mut`를 붙여야 한다.
  * 이 기본값 덕분에 값이 어디서 바뀌는지 추적하기 쉬워진다.

```rust
let a = 1;     // immutable
let mut b = 2; // mutable

let ra = &a;     // immutable borrow
let rb = &mut b; // mutable borrow
```

* immutable로 선언한 값을 나중에 `&mut`로 빌릴 수는 없다.
  * `&mut a`는 사용할 수 없다.
  * mutable borrow를 만들려면 원래 binding이 `let mut`이어야 한다.
* 불변으로 선언한 값을 바꿔야 하는 일이 생겼다면 선택지는 두 가지다.
  * 실제로 값이 변해야 하는 모델이라면 처음부터 `let mut`으로 선언한다.
  * 기존 값을 유지하고 새 값을 만들면 되는 상황이라면 shadowing으로 새 binding을 만든다.
* 즉 Rust에서 중요한 질문은 "이 값이 변해야 하는가, 아니면 새 값을 만들어도 되는가"다.
