* Arduino에서 [ESP-01 모듈](https://www.devicemart.co.kr/goods/view?no=1279338)에 AT command를 보내 HTTP GET 요청을 테스트했다.
  * 전체 흐름은 TCP 연결 생성, 보낼 바이트 수 선언, HTTP 요청 전송, 연결 종료 순서다.
* [AT+CIPSTART 명령어](https://docs.espressif.com/projects/esp-at/en/latest/esp32/AT_Command_Set/TCP-IP_AT_Commands.html#cmd-start)를 사용해서 서버에 접속한다.
    * `AT+CIPSTART="TCP","ADDRESS",POST`
  * 여기서 마지막 값은 HTTP method의 `POST`가 아니라 TCP port 번호다.
* [AT+CIPSEND 명령어](https://docs.espressif.com/projects/esp-at/en/latest/esp32/AT_Command_Set/TCP-IP_AT_Commands.html#cmd-send)를 사용해서 몇 바이트를 보낼 것인지 명시한다.
    * `AT+CIPSEND=BYTE_SIZE`
      * 이때 데이터의 제일 마지막에 CRLF가 들어가는 것을 기억해서 +2 해줘야 한다.
    * 이후에 `OK >`가 나오면 데이터를 입력한다.
      * 이번에 보내는 데이터는 http 요청의 가장 기본적인 내용 `GET / HTTP/1.1\r\n`로 16바이트이다.
    * 전송이 무사히 끝났다면 `SEND OK`가 모듈에서 반환된다
  * HTTP/1.1 요청이라면 원래 `Host` 헤더와 헤더 종료를 위한 빈 줄까지 보내는 편이 맞다.
  * 여기서는 Python `http.server`가 단순 요청을 받아줘서 최소 형태로도 응답을 확인할 수 있었다.
* [AT+CIPCLOSE 명령어](https://docs.espressif.com/projects/esp-at/en/latest/esp32/AT_Command_Set/TCP-IP_AT_Commands.html#cmd-close)를 사용해서 접속을 종료한다.
    * 여러개의 접속을 사용하면 link-id를 입력해야 한다.
    * 지금은 한개의 접속만 존재하기 때문에 상관없다.
* 시리얼 모니터로 직접 명령어를 전송한 기록이다.
    ```text
    00:47:38.598 -> AT+CIPSTART="TCP","192.168.0.2",9999
    
    00:47:38.641 -> CONNECT
    00:47:38.641 -> 
    00:47:38.641 -> OK
    00:47:47.507 -> AT+CIPSEND=16
    
    00:47:47.549 -> 
    00:47:47.549 -> OK
    00:47:47.549 -> > 
      GET / HTTP/1.1
    
    00:47:53.481 -> Recv 16 bytes
    00:47:53.481 -> 
    00:47:53.481 -> SEND OK
    00:49:15.455 -> AT+CIPCLOSE
    
    00:49:15.501 -> CLOSED
    00:49:15.501 -> 
    00:49:15.501 -> OK
    ```

* 서버 로그에서도 GET 요청이 들어온 것을 확인할 수 있다.
    ```text
    
    ➜  ~ python3 -m http.server 9999
    Serving HTTP on 0.0.0.0 port 9999 (http://0.0.0.0:9999/) ...
    192.168.0.5 - - [29/Oct/2022 00:49:15] "GET / HTTP/1.1" 200 -
    ```
