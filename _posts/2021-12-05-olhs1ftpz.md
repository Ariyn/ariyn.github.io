Docker 이미지에 `ENTRYPOINT`가 설정되어 있는데 임시로 shell만 열고 싶을 때 `--entrypoint`를 사용한다.
예시는 `docker run -it --entrypoint /bin/bash <image>`다.

{: .tractatus}
1. Docker 이미지에 `ENTRYPOINT`가 설정되어 있는데 임시로 shell만 열고 싶을 때 `--entrypoint`를 사용한다.

   1. 예시는 `docker run -it --entrypoint /bin/bash <image>`다.

   2. 이미지에 `bash`가 없다면 `/bin/sh`를 사용한다.

2. 이 옵션은 이미지에 정의된 기본 entrypoint를 실행하지 않고, 지정한 명령을 대신 실행하게 한다.

   1. 컨테이너 시작 스크립트가 실패해서 내부를 확인해야 할 때 특히 편하다.

   2. 환경 변수, 파일 위치, 권한 문제를 직접 확인할 수 있다.

3. 단, 이 방식은 디버깅용으로 쓰는 것이 좋다.

   1. 실제 실행 경로를 검증하려면 원래 entrypoint로도 다시 실행해 봐야 한다.
