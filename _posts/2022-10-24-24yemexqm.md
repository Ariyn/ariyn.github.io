`git clone`만 실행하면 submodule 내용까지 자동으로 내려받지 않는다.
부모 저장소에는 submodule의 경로와 커밋 정보만 들어 있다.

{: .tractatus}
1. `git clone`만 실행하면 submodule 내용까지 자동으로 내려받지 않는다.

   1. 부모 저장소에는 submodule의 경로와 커밋 정보만 들어 있다.

   2. 실제 submodule 작업 디렉터리는 별도로 초기화하고 가져와야 한다.

2. 처음 clone할 때 한 번에 가져오려면 `--recursive`를 붙인다.

   ```shell
   git clone --recursive https://github.com/some/repo.git
   ```

3. 이미 clone한 뒤라면 아래 명령으로 submodule을 가져올 수 있다.

   ```shell
   git submodule update --init --recursive
   ```

4. submodule을 쓰는 프로젝트에서 빌드 파일이 없거나 디렉터리가 비어 보이면 이 부분부터 확인하는 것이 좋다.
