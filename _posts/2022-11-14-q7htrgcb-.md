* `wkhtmltopdf`에서 page break를 제어하려면 HTML/CSS 쪽에 명시적인 class를 두는 방식이 편하다.
  * markdown 문서 안에서는 `<p class="break-before"></p>`처럼 빈 HTML 요소를 넣어 page break 위치를 표시한다.
  * stylesheet에는 해당 class의 page break 규칙을 정의한다.

```css
.keep-together {
    page-break-inside: avoid;
}

.break-before {
    page-break-before: always;
}

.break-after {
    page-break-after: always;
}
```

* `break-before`는 해당 요소 앞에서 페이지를 나누고, `break-after`는 뒤에서 나눈다.
  * 표나 코드블록처럼 중간에 끊기면 안 되는 요소에는 `page-break-inside: avoid`를 적용한다.
* PDF 생성은 HTML 렌더링 엔진의 제약을 많이 받는다.
  * markdown에서 보이는 구조와 PDF 페이지 나눔은 별도 문제라, 최종 PDF를 열어 확인해야 한다.
