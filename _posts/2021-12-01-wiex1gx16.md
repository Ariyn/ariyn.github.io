특정 커밋 이후부터 현재 `HEAD`까지의 로그를 보고 싶다면 range 표기를 사용한다.
기본 형태는 `git log <since_commit>..HEAD`다.

{: .tractatus}
1. 특정 커밋 이후부터 현재 `HEAD`까지의 로그를 보고 싶다면 range 표기를 사용한다.

   1. 기본 형태는 `git log <since_commit>..HEAD`다.

   2. 이 경우 `<since_commit>` 자체는 결과에 포함되지 않는다.

2. 기준 커밋까지 포함하고 싶다면 부모 커밋을 왼쪽 기준으로 둔다.

   1. `git log <since_commit>^..HEAD`처럼 쓴다.

   2. `^`는 해당 커밋의 부모를 의미한다.

3. Git range는 왼쪽은 제외 기준, 오른쪽은 포함 방향이라고 생각하면 이해하기 쉽다.

   1. 릴리스 노트나 변경 목록을 뽑을 때 자주 쓰는 패턴이다.
