* logrotate는 계속 떠 있는 데몬이라기보다, 정해진 주기에 한 번 실행되는 배치 작업에 가깝다.
  * 그래서 배포판과 init 시스템에 따라 `cron.daily`나 `systemd timer`로 실행된다.
  * 예전 환경이나 systemd가 없는 환경에서는 cron이 `/etc/cron.daily/logrotate`를 호출하는 구조를 볼 수 있다.
* `/etc/cron.daily/logrotate`의 내용은 대략 아래처럼 되어 있다.
  ```shell
    #!/bin/sh
    
    # skip in favour of systemd timer
    if [ -d /run/systemd/system ]; then
        exit 0
    fi
    
    # this cronjob persists removals (but not purges)
    if [ ! -x /usr/sbin/logrotate ]; then
        exit 0
    fi
    
    /usr/sbin/logrotate /etc/logrotate.conf
    EXITVALUE=$?
    if [ $EXITVALUE != 0 ]; then
        /usr/bin/logger -t logrotate "ALERT exited abnormally with [$EXITVALUE]"
    fi
    exit $EXITVALUE
  ```

  * 첫 줄의 `skip in favour of systemd timer`가 중요하다.
  * systemd가 동작 중이면 cron job은 바로 종료하고, 실제 실행은 `logrotate.timer` 쪽에 맡긴다.
* cron으로 실행되는 환경에서는 `/etc/crontab`의 `cron.daily` 항목이 실행 시각을 결정한다.
  ```shell
    25 6	* * *	root	test -x /usr/sbin/anacron || ( cd / && run-parts --report /etc/cron.daily )
  ```

  * 이 경우 `/etc/cron.daily/logrotate`가 매일 6시 25분에 실행된다.
  * 다만 anacron이 설치되어 있거나 systemd timer를 쓰는 배포판에서는 실제 실행 시각이 달라질 수 있다.
* 확인할 때는 둘을 같이 보면 된다.
  * cron 경로: `cat /etc/cron.daily/logrotate`, `cat /etc/crontab`
  * systemd 경로: `systemctl list-timers logrotate.timer`, `systemctl cat logrotate.timer`
